vue工程化

发表于 2021-11-09 更新于 2021-11-12 分类于 Vue
本文字数： 2.1k 阅读时长 ≈ 2 分钟

vue-cli 3.0以上才会自带 vue ui ；vue create ‘项目名’

1.单页面应用程序

在工程化的项目中，vue要做的事情就是，通过main.js 将App.vue(各种vue组件)，渲染到index.html的指定区域中。

2.运行项目

package.json 中 “scripts”： “serve” 开发 ；“build” 发布

npm run serve, 在终端启动项目

在vue ui 中比较好操作！！！

3.根组件问题

render函数中，渲染的是哪个 .vue组件，这个组件就是根组件。

（其他的组件都放在components这个文件夹下）

el:’#app’/ .$mount(‘#app’)，谁挂载到单页面应用index.html中，谁就是根组件。

在一般工程化项目中，import App from App.vue；这个App.vue就是根组件。

4.vue组件的三个组成部分

一键生成标三部分格式：先要装插件，然后输入<, 然后选择第一个，就能一键生成

  • templates 组件的模板结构（必须有，里面有一个最大的<div> </div>）

  • // 固定写法 export default
export default{
    // 这里的 data 是一个函数，这样在多个组件的时候，就不会相互影响。（如果是对象就会有问题）
    data(){
        return {
            username:'',
            age:''
        }    
    },
    methods:{},   
    watch:{}, // 监视器 handler(newVal){}
    computed:{}, // 复用的计算方法放在这里面
}

通过components注册的是私有子组件，在组件A的components节点下，就只能在组件A中使用

6.全局注册组件

在 vue 项目中 main.js 入口文件中，通过 Vue.components()方法，可以注册全局组件。

7.组件的props

props是组件的自定义属性，在封装通用组件的时候，合理地使用 props 可以极大的提高组件的复用性！

props允许使用者通过自定义属性，为当前组件指定初始值。

props的值是只读的，可以用插值表达式 放在全局组件的模块结构中，就是可以渲染出来，显示到页面，但不要去修改。不同的其他组件使用这个全局组件，然后可以通过在 标签 <MyCount init=”9” ></MyCount> 这样来设置。

8.组件之间的样式冲突问题

默认情况下，写在.vue 组件中的样式会全局生效，因此很容易造成多个组件之间的样式冲突问题。

导致组件之间样式冲突的根本原因是：

  • 单页面应用程序中，所有组件的DOM结构，都是基于唯一的 index.html 也面进行呈现的
  • 每个组件中的样式，都会影响整个 index.html 页面中的DOM元素

9. .vue会被解析会js文件

浏览器不会解析.vue文件的，package.json文件中有一个配置：“vue-template-compiler” ：“ 2.6.11 ”，会将.vue文件转换成 JavaScript 文件让浏览器解析

10.组件的生命周期

生命周期：是指一个组件从 *创建 – 渲染 – 运行 – 销毁 *的整个过程

生命周期函数：是由vue 框架提供的内置函数，会伴随着组件的生命周期，自动调用执行

new Vue()； 每个vue组件 就是一个vue实例

【webpack 从 main.js开始打包，发现main.js中用到了App.vue；App.vue又用到了Left.vue，Right.vue；后面俩组件又用到了Count.vue；这样一步步往后面延伸，一步步的开启组件的生命周期（ps: 组件在封装好没调用之前，都没有进入生命周期中，即没有实例new，只是封装好了）】

  1. 父传子。使用props[‘init’] 自定义属性。子组件 props 中的值，可以用插值表达式写在模板结构中。子组件不要去修改props里值的数据，拿来用就好。一般是父组件传给子组件的值。

  1. 子传父。使用 $emit(‘xxx’,this.count)。自定义事件。

  1. 兄弟组件之间传值。
